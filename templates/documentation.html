Data Exploration and Preprocessing Documentation

As it can be seen on the Data Exploration page, there are many features that can be used to predict wine quality. I obtained the wine quality dataset from this link. Below I mention the steps I took for Data Cleaning and Preprocessing:

  • The dataset contained two different files for red wine and white wine. I combined both the files and added another column to signify whether the wine was red or white.
  • I removed all the data that contained empty features. And at the end I had a working dataset of 6914 elements.
  • The quality points are on the scale of 1-10. But as you can see in the data visualizations, the ratings in the dataset are only in the range of 3-8. This creates some confusion about the meaning of the ratings. Hence, I decided to make things simple by having ratings 1-5 signify low quality wine and ratings 6-10 signify high quality wine.
  • Then I analyzed the distribution of high and low quality wine in the dataset. Turns out it’s almost equal and hence, we don’t face the issue of data imbalance.
  • Next, I analyzed all the features. For preprocessing them and to ensure that there are no skewed features, I used mean normalization on all the features, by subtracting each data point by the mean of the feature and dividing it by it’s standard deviation.
  • So now we have a classification problem of predicting whether the wine is ‘high’ quality or ‘low’ quality.

Predictive Machine Learning Model Documentation

After I cleaned and preprocessed the data, I move on to explaining the Machine Learning model I used to predict the quality of wine based on the features I extracted.

  • For this classification task, I experimented with two Machine Learning models: Support Vector Machine and Random Decision Forest.
  • I experimented with a few parameters and noted the F1 score for each experiment. Then I ran a grid search to find the most optimal set of hyper parameters.
  • I used cross-validation by randomly splitting my dataset into train set and validation set to correctly evaluate the model performance.
  • The random forest gave the best performance from all the experiments I tried with F1 score of 0.81 and accuracy 0.82.
